# <a name="permissions-and-public-folders"></a>Le autorizzazioni e le cartelle pubbliche

È possibile modificare le autorizzazioni delle cartelle pubbliche utilizzando Outlook, l'interfaccia di amministrazione di Exchange (EAC) o PowerShell:
  
- Per istruzioni di Outlook, [fare clic qui](https://support.office.com/article/https://support.office.com/article/Set-or-change-permissions-for-a-public-folder-b2e0440c-7873-48ec-9ff2-b1a20b723005.aspx).
    
- Per EAC, fare riferimento a [questo articolo](https://support.office.com/article/https://technet.microsoft.com/library/jj651147%28v=exchg.150%29.aspx.aspx#Anchor_1) per istruzioni. È possibile fare clic [qui](https://support.office.com/article/ https://outlook.office365.com/ecp/.aspx) per passare a EAC. 
    
- Per Powershell, fare riferimento a [questo articolo](https://support.office.com/article/https://technet.microsoft.com/library/bb124743%28v=exchg.160%29.aspx.aspx) per istruzioni su come utilizzare il cmdlet Add-PublicFolderClientPermission. Se è necessario istruzioni per la connessione a Exchange Powershell, fare clic [qui](https://support.office.com/article/https://technet.microsoft.com/library/jj984289%28v=exchg.160%29.aspx.aspx).
    
Se **gli utenti esterni non possono inviare messaggi di posta elettronica a una cartella pubblica abilitata alla posta elettronica**, la causa potrebbe essere che la cartella pubblica non dispone delle autorizzazioni necessarie per il recapito di posta elettronica esterno. È possibile risolvere il problema utilizzando le istruzioni fornite Outlook [qui](https://support.office.com/article/https://technet.microsoft.com/library/aa997560%28v=exchg.150%29.aspx.aspx#Anchor_1)o le istruzioni di PowerShell [di seguito](https://support.office.com/article/https://support.microsoft.com/help/2984402/-5.7.1-smtp-550-5.7.1-resolver.rst.authrequired-nondelivery-report-when-external-users-try-to-send-mail-to-mail-enabled-public-folders-in-office-365.aspx).
